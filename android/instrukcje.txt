Functional spec you must implement across the project:

1. SOURCE MIXER
Three main input channels:
- VIDEO
- MUSIC
- MIC

For each channel:

- Mute toggle
- Solo toggle (soloing one channel mutes others)
- Bypass toggle (bypasses FX chain for this source if applicable)
- Channel meters for level (peak / RMS)

VIDEO channel
- Source can be:
  - camera rear
  - camera front
  - video file (local)
- Transport controls for video files:
  - Play / Pause
  - Previous / Next
  - Seek (basic is enough at first)
- Scale / crop control integrated with Output & Framing section.

MUSIC channel
- Source can be:
  - local audio file (MP3 / WAV)

  - Play / Pause
  - Previous / Next
- Level meter.

MIC channel
- Source: device microphone
- Gain control
- Mute / Unmute
- Basic noise floor handling (simple gate or low cutoff is enough for MVP).

Routing
- There must be a clear notion of which audio source is used for analysis (FFT):
  - user can pick: MUSIC, MIC, VIDEO audio, or a MIX from the mixer.

2. AUDIO ANALYSIS AND SPECTRUM
- Use FFT (2048 or 4096) to compute spectrum from the selected audio source.
- Derive at least these normalized bands:
  - BASS: 20 – 200 Hz
  - MID: 200 – 2000 Hz
  - HIGH: 2000 – 8000 Hz
- For each band keep:
  - instantaneous value
  - smoothed value (attack and release)
- Provide peak detection for rhythmic triggers.

Interactive spectrum UI:
- A visual area with three colored circles or nodes for BASS, MID and HIGH.
- Each node shows current band magnitude (radius and intensity).
- This view is updated in real time and feels smooth.

4. FX CONTROLS – MAIN FX PANEL

Controls (generic set, exact mapping is per shader):
- INTENSITY
- SCALE / ZOOM
- DISTORT AMOUNT
- SPEED
- OFFSET X
- OFFSET Y
- COLOR SHIFT
- EXPOSURE / BRIGHTNESS
- CONTRAST
- BLUR / SHARPNESS
- GLITCH AMOUNT / MODE

UI for FX controls:

5. BPM & SYNC
- A small BPM section:
  - BPM value (manual input and slider)

- Provide GLSL uniform values:
  - uBPM
  - uBeatPhase or similar

6. FX CHAIN
FX chain structure:
- Header FX
- Layer 1
- Layer 2
- Layer 3
- Layer 4

For each FX slot:
- Dropdown with preset name (or “None”)
- Two or more knobs for key parameters of that FX
- Blend mode selector:
  - NORMAL
  - ADD
  - MULTIPLY
  - SCREEN
  - LIGHTEN
  - DARKEN (if simple to implement)

Routing:
- Source → Header FX → Layer 1 → Layer 2 → Layer 3 → Layer 4 → Output
- Layers can either:
  - operate serially in the pipeline
  - or be rendered separately and blended (you decide, but keep it efficient).

7. GLSL SHADERS AND UNIFORMS
Target:
- Migrate and support my existing web shaders from SHADER_LIST.

Each shader:
- Written in GLSL ES 3.0, compatible with OpenGL ES 3.0.
- Uses a Shadertoy like header with:
  - uniform float iTime
  - uniform vec2 iResolution
  - camera or video source as samplerExternalOES or sampler2D
  - uniforms for:
    - uBass, uMid, uHigh
    - uIntensity
    - uScale
    - uOffsetX, uOffsetY
    - uColorShift
    - uGlitch, uGlitchMode
    - uBPM, uPhase
    - mixStrength or similar.

Implement a shader manager:
- Dynamically compiles and links shaders.
- Provides a typesafe way in Kotlin to set uniforms each frame.
- Supports switching presets at runtime with minimal hitches.
- Loads shader bodies from resources (raw/assets) if needed.

9. RECORDING PIPELINE – MEDIA CODEC + EGL
Goal:
- Record the final GL output along with audio to an MP4 file.

Constraints:
- No glReadPixels
- Use dual surface rendering:
  - one EGLSurface for onscreen rendering (GLSurfaceView)
  - one EGLSurface bound to MediaCodec input Surface.

Implementation plan:
- Create a VisusRecorder module/class that:
  - configures a H.264 encoder with:
    - resolution according to selected aspect ratio and chosen output size
    - bit rate around 10–15 Mbps for 1080p (you can adjust based on performance)
    - 30 or 60 FPS
  - creates a MediaMuxer for MP4 output
  - exposes:
    - startRecording(config)
    - stopRecording()
    - updateOnFrame() or similar hook.

- In the GL renderer:
  - when recording is active, draw each frame twice:
    - once to the onscreen EGLSurface
    - once to the encoder EGLSurface
  - handle presentation timestamps properly.

- Audio for recording:
  - For now, pick one audio source (e.g. MUSIC or mixed bus)
  - Encode using AAC and mux with the video stream.

UI:
- Record button:
  - toggles recording state
  - shows clear visual state (e.g. red glowing when recording)
- On stop:
  - the app saves an MP4 file in a user visible folder (e.g. DCIM/VISUS or Movies/VISUS)
  - shows a toast or snackbar with the path.
